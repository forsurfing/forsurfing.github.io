## 加法乘法

- 加法乘法两种普通时间复杂度不可能低于O ($O(n^2)$)
- 但是如果用分治思想，也就是（KARATSUBA CUTOFF算法）算法，复杂度可以降到 ($O(n\log n)$)

### 1. 基本情况
- n ≤ cutoff: O(n²) 普通乘法
- n > cutoff: 递归过程

### 2. 递归情况分析
- 主定理：T(n) = 3T(n/2) + O(n)
- 分为3个子问题，每个规模n/2
- 合并需要O(n)操作
- 形成公式：T(n) = 3T(n/2) + cn

### 3. 递归树展开
- 第0层：cn
- 第1层：3c(n/2)
- 第2层：9c(n/4)
- 第3层：27c(n/8)
- 第k层：3^k c(n/2^k)

### 4. 总复杂度
O(n^(log₂3)) ≈ O(n^1.585)
- log₂3 ≈ 1.585 
- 这比普通乘法的O(n²)更优
- 但实际中只有当n大于某个cutoff值时才有优势
